# 数据库设计文档

数据库：**SQLite3**　文件位置：`data/autism.db`（首次启动自动创建）
字符集：UTF-8　时间字段：本地时间 `datetime('now','localtime')`

依据项目要求设计 **6 张核心表** + **3 张扩展表**（科普文章、转诊机构、赋能资源）。

---

## 一、核心表

### 1. users（用户表）
| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK AUTOINCREMENT | 主键 |
| phone | VARCHAR(20) | UNIQUE NOT NULL | 手机号 |
| code | VARCHAR(10) |  | 最近一次验证码 |
| password | VARCHAR(100) |  | BCrypt 加密密码（验证码登录可空） |
| nickname | VARCHAR(50) |  | 昵称（微信登录使用） |
| openid | VARCHAR(64) |  | 微信 openid |
| avatar | VARCHAR(255) |  | 头像 URL |
| create_time | DATETIME | DEFAULT now | 创建时间 |
| update_time | DATETIME | DEFAULT now | 更新时间 |

### 2. children（儿童表）
| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK AUTOINCREMENT | 主键 |
| user_id | INTEGER | NOT NULL, FK→users(id) | 所属用户 |
| name | VARCHAR(50) | NOT NULL | 昵称 |
| gender | VARCHAR(10) | NOT NULL | male / female |
| birth_date | DATE | NOT NULL | 出生日期（年龄限制 1-5 岁） |
| avatar | VARCHAR(20) |  | 头像 emoji |
| create_time | DATETIME | DEFAULT now | 创建时间 |

### 3. questionnaires（问卷表）
| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK AUTOINCREMENT | 主键 |
| title | VARCHAR(100) | NOT NULL | 问卷标题 |
| description | TEXT |  | 问卷说明 |
| total_questions | INTEGER | DEFAULT 20 | 题目总数 |
| create_time | DATETIME | DEFAULT now | 创建时间 |

### 4. questions（题目表，含视频）
| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK AUTOINCREMENT | 主键 |
| qid | INTEGER | NOT NULL, FK→questionnaires(id) | 所属问卷 |
| video_url | VARCHAR(255) |  | 讲解视频地址 |
| content | TEXT | NOT NULL | 题干 |
| options | TEXT | NOT NULL | 选项 JSON：`[{value,label,score}]` |
| sort | INTEGER | NOT NULL | 排序 |

> **反向计分**：第 12 题（刻板行为）选项 `value` 与 `score` 反向对应，score 在服务端按题库重新计算，防止前端篡改。

### 5. answers（答卷表，含报告）
| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK AUTOINCREMENT | 主键 |
| child_id | INTEGER | NOT NULL, FK→children(id) | 筛查儿童 |
| qid | INTEGER | NOT NULL, FK→questionnaires(id) | 问卷 |
| answer_json | TEXT | NOT NULL | 答案 JSON：`[{questionId,value,label,score}]` |
| total_score | INTEGER | NOT NULL | 总得分 |
| risk_level | VARCHAR(10) | NOT NULL | low / medium / high |
| create_time | DATETIME | DEFAULT now | 筛查时间 |

### 6. admins（管理员表）
| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK AUTOINCREMENT | 主键 |
| username | VARCHAR(50) | UNIQUE NOT NULL | 用户名 |
| password | VARCHAR(100) | NOT NULL | BCrypt 加密 |
| create_time | DATETIME | DEFAULT now | 创建时间 |

> 默认管理员：`admin / admin123`（首次启动由 `DataInitializer` 自动创建）。

---

## 二、扩展表

### 7. articles（科普文章）
`id, category(knowledge/early/intervene), title, summary, content, author, date, create_time`

### 8. institutions（转诊机构）
`id, name, region, department, address, phone, description, create_time`

### 9. resources（赋能资源）
`id, icon, title, description, url, create_time`

---

## 三、索引
- `idx_children_user`（children.user_id）
- `idx_questions_qid`（questions.qid）
- `idx_answers_child`（answers.child_id）
- `idx_answers_risk`（answers.risk_level）

## 四、外键与级联
所有外键均配置 `ON DELETE CASCADE`：删除用户级联删除其儿童 → 级联删除其答卷。
SQLite 默认不开启外键约束，Spring Boot 启动时已通过 `PRAGMA foreign_keys=ON` 启用（如需启用可在 JDBC URL 追加 `?foreign_keys=on`）。

## 五、评分与风险判定规则
- 每题 0-3 分，总分 0-60 分
- 🟢 低风险 low：0-15 分
- 🟡 中风险 medium：16-35 分
- 🔴 高风险 high：36-60 分

## 六、初始化数据
`schema.sql` 建表，`data.sql` 幂等写入种子数据：
- 1 份 SACS 改编问卷 + 20 道题目
- 8 篇科普文章、12 家转诊机构、8 项赋能资源
- 管理员账号由 Java 代码 `DataInitializer` 写入（密码 BCrypt 加密）
